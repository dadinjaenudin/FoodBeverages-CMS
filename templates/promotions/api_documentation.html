Authentication

All API requests require token-based authentication.

// Request Header
Authorization: Token YOUR_API_TOKEN

Security Note

Keep your API token secure. Do not commit it to version control.

Get Promotions for Store

Endpoint

GET /api/v1/sync/promotions/

Query Parameters

Parameter Type Required Description
store_id UUID Required Store UUID to get promotions for
company_id UUID Required Company UUID
brand_id UUID Optional Brand UUID to filter promotions
updated_since ISO DateTime Optional Only return promotions updated after this timestamp (for incremental sync)

Example Request

// Full sync (all promotions)
curl -X GET "{{ request.scheme }}://{{ request.get_host }}/api/v1/sync/promotions/ \
?store_id={% if sample_store %}{{ sample_store.id }}{% else %}YOUR_STORE_UUID{% endif %} \
&company_id={{ request.user.company.id }}" \
-H "Authorization: Token YOUR_API_TOKEN"

// Incremental sync (only updated promotions)
curl -X GET "{{ request.scheme }}://{{ request.get_host }}/api/v1/sync/promotions/ \
?store_id={% if sample_store %}{{ sample_store.id }}{% else %}YOUR_STORE_UUID{% endif %} \
&company_id={{ request.user.company.id }} \
&updated_since=2026-01-27T10:00:00Z" \
-H "Authorization: Token YOUR_API_TOKEN"

Error Handling

HTTP Status Codes

200 Success - Promotions returned successfully
400 Bad Request - Missing required parameters
401 Unauthorized - Invalid or missing token
404 Not Found - Store not found
500 Server Error - Internal server error

Error Response Format

{
  "error": "Store not found",
  "code": "STORE_NOT_FOUND",
  "timestamp": "2026-01-27T10:00:00Z"

üîÑ Typical Sync Flow for Edge Server
  1. Initial Setup: GET /api/v1/sync/companies/ ‚Üí Get all companies
  2. Select Company: User selects company from dropdown
  3. Load Brands: GET /api/v1/sync/brands/?company_id={uuid} ‚Üí Get brands for selected company
  4. Select Brand: User selects brand from dropdown
  5. Load Stores: GET /api/v1/sync/stores/?brand_id={uuid} ‚Üí Get stores for selected brand
  6. Select Store: User selects store (this becomes the Edge Server identity)
  7. Sync Data: Use store_id to sync promotions, products, etc.
